Management คือ การจัดการระบบข้อมูล มีหน้าที่รับผิดชอบ สำหรับการปรับปรุงเเละจัดระเบียบ หรือ เป็นที่รู้ๆกันว่าเป็นซอฟเเวร์คอม ที่มีหน้าที่รับผิดชอบในการจัดการ กลุุ่มไฟล์ รวมถึงเอกสารบันทึกข้อมูลที่มีการถูกจำกัด

เพราะฉะนั้น File Management คือ คือกระบวนการจัดระเบียบ จัดเก็บ ตั้งชื่อ และดูแลรักษาข้อมูลดิจิทัลบนอุปกรณ์คอมพิวเตอร์หรือคลาวด์อย่างเป็นระบบ เพื่อให้สามารถค้นหา เรียกใช้งาน แก้ไข และสำรองข้อมูลได้อย่างง่ายดาย รวดเร็ว และปลอดภัย ลดปัญหาไฟล์สูญหาย

File มี 2 ประเภท คือ 
Text Files (Plain Text) เเละ Binary Files

1. ไฟล์ข้อความ (Text File): ไฟล์ที่เก็บข้อมูลในรูปแบบตัว  อักษร ASCII หรือ UTF-8 สามารถเปิดอ่านได้ด้วยโปรแกรม Text Editor ทั่วไป เช่น Notepad, TextEdit

2. ไฟล์ฐานข้อมูลหรือไฟล์ไบนารี (Binary File): ไฟล์ที่เก็บข้อมูลเป็นรหัส 0 และ 1 ที่คอมพิวเตอร์เข้าใจ มักต้องใช้โปรแกรมเฉพาะทางในการเปิด เช่น ไฟล์รูปภาพ, ไฟล์เพลง, หรือไฟล์ฐานข้อมูล

โครงสร้างพื้นฐานของไฟล์ 
(File Architecture Structure)
โดยทั่วไปไฟล์ข้อมูลจะมีโครงสร้างเป็นลำดับ
1. Field (ฟิลด์)
หน่วยข้อมูลที่เล็กที่สุด
2. Record (เรคคอร์ด)
กลุ่มของ Field ที่รวมกันเป็นข้อมูลของหนึ่งรายการ
3. File (แฟ้มข้อมูล)
กลุ่มของ Record หลายรายการ
4. Database (ฐานข้อมูล)
กลุ่มของไฟล์หลายไฟล์ที่เกี่ยวข้องกัน

รูปแบบการจัดโครงสร้างไฟล์ (Types of File Architecture)
1. Sequential File Structure
ข้อมูลถูกเก็บ เรียงตามลำดับ
2. Direct / Random Access File
สามารถเข้าถึงข้อมูล ตำแหน่งใดก็ได้โดยตรง
3. Indexed File
มี index (ดัชนี) ช่วยค้นหาข้อมูล

The Architecture of a file System ระบบไฟล์เป็นส่วนสำคัญ ทำหน้าที่ จัดโครงสร้างสำหรับการจัดเก็บ จัดระเบียบ และจัดการข้อมูลบนอุปกรณ์เก็บข้อมูล เช่น ฮาร์ดดิสก์(SSD , USB Drive)
